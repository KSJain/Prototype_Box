# KSJ Engineering Standard

## Purpose

Build systems that are reliable, understandable, and enjoyable to maintain.

Technology should feel less like operating a machine and more like interacting with a thoughtful companion.

---

# Engineering Principles

Safety before features.

Observe before acting.

Automate neglect, not relationships.

Local first.

Non-blocking architecture.

Small consistent progress.

Hardware should be replaceable.

Software should be understandable.

If uncertain,
STOP.

If it cannot be tested,
it is not finished.

If it cannot be understood,
simplify it.

If it cannot be maintained,
redesign it.

Take one.
Give back two.

---

# Architecture

Prefer event-driven systems.

Prefer state machines over delay().

Prefer composition over duplication.

Libraries solve one problem.

Products compose libraries.

Separate hardware from business logic.

Separate business logic from presentation.

Design for replacement.

Fail safely.

---

# Coding Guidelines

Prefer constexpr.

Prefer enum class.

Prefer const correctness.

Prefer meaningful names.

Prefer namespaces over globals.

Keep APIs small.

Keep classes focused.

Keep functions short.

Avoid hidden state.
